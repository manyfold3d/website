This release adds a couple of new features, and a few bugfixes, as well as some internal refactoring and improved tests. You can now set a preferred "landing page", like "my models" or "all creators", as well as the current "dashboard" - and that can be site-wide, or per-user. And, due to some of that internal refactoring, we now have syntax highlighting for some text files (e.g. Javascript).

## What's Changed
### ✨ New Features ✨
* Add site-wide and user-specific setting for default landing page by [@Floppy](https://github.com/Floppy) in [#6133](https://github.com/manyfold3d/manyfold/pull/6133)
* Add syntax highlighting for some files using Rouge by [@Floppy](https://github.com/Floppy) in [#6169](https://github.com/manyfold3d/manyfold/pull/6169)
### 🐛 Bug Fixes 🐛
* Prevent escaping from library setup on first run by [@Floppy](https://github.com/Floppy) in [#6146](https://github.com/manyfold3d/manyfold/pull/6146)
* Fix error when showing collections for federated items by [@Floppy](https://github.com/Floppy) in [#6162](https://github.com/manyfold3d/manyfold/pull/6162)
* Allow multiple spaces in filenames by [@Floppy](https://github.com/Floppy) in [#6184](https://github.com/manyfold3d/manyfold/pull/6184)
