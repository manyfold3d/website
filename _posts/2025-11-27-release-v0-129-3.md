Another patch release, this time focused on security improvements from our recent audit. There are also a few bugfixes, including an important one from the last release that potentially breaks model pages for new users!

## What's Changed
### üîí Security üîí
* Obfuscate password input fields in user admin area by [@Floppy](https://github.com/Floppy) in [#5094](https://github.com/manyfold3d/manyfold/pull/5094)
* Sanitize upload filenames to prevent path traversal by [@Floppy](https://github.com/Floppy) in [#5098](https://github.com/manyfold3d/manyfold/pull/5098)
* Only object owners can set sharing permissions by [@Floppy](https://github.com/Floppy) in [#5099](https://github.com/manyfold3d/manyfold/pull/5099)
* Obfuscate OAuth client secret on screen (with reveal and copy options) by [@Floppy](https://github.com/Floppy) in [#5100](https://github.com/manyfold3d/manyfold/pull/5100)
* OIDC: Don't match accounts by unverified emails by [@Floppy](https://github.com/Floppy) in [#5101](https://github.com/manyfold3d/manyfold/pull/5101)
* Improve and test rate limiting, including OAuth and OIDC endpoints by [@Floppy](https://github.com/Floppy) in [#5104](https://github.com/manyfold3d/manyfold/pull/5104)
* Add explicit sanitization to fields that come in from the Fediverse by [@Floppy](https://github.com/Floppy) in [#5111](https://github.com/manyfold3d/manyfold/pull/5111)
